**Note:** There is no `module_index` table. The DB has no dedicated module index table. Module nodes are stored directly in the `nodes` table with `kind='Module'` and `payload` = absolute file path. The relevant tables are `nodes` (all graph nodes), `node_references` (reference edges), `transactions`, and `operations` (operation log). Vector tables (`node_embeddings`, `commit_pattern_embeddings`) and their virtual-table shadow tables are also present.

Full schema:

```sql
CREATE TABLE nodes (
      id TEXT PRIMARY KEY,
      kind TEXT NOT NULL,
      parent_id TEXT,
      child_index INTEGER,
      payload TEXT
    );
CREATE INDEX nodes_parent_kind_idx ON nodes(parent_id, kind);
CREATE TABLE node_references (
      from_node_id TEXT NOT NULL PRIMARY KEY,
      to_node_id   TEXT NOT NULL,
      kind         TEXT NOT NULL,
      FOREIGN KEY (from_node_id) REFERENCES nodes(id),
      FOREIGN KEY (to_node_id)   REFERENCES nodes(id)
    );
CREATE INDEX node_references_to_idx ON node_references(to_node_id);
```

### Task 5 — commit-gate sanity (no-op tx)

**Command:**
```bash
npx tsx scripts/l2.5-prep-noop-commit.ts /tmp/valibot.strata.db "$HOME/code/valibot-l2.5/library" 2>&1 | tee /tmp/valibot-noop-commit.log
```

**Result:** `ok: false`, 531 diagnostics.

**Conclusion:** The commit gate surfaces a Strata-side scoping gap: `validate()` in `commit()` uses `rootNames: [...renderedFiles.keys()]`, which includes ALL ingested modules — test files (`*.test.ts`, `*.test-d.ts`), root-level tool configs (`tsdown.config.ts`, `vitest.config.ts`, `playground.ts`), and source files — rather than scoping to src-only files. This causes `vitest` (a test-only import) and `tsdown` (a dev tool) to be required in the tsc program, triggering 531 "cannot find module" / `isolatedDeclarations` errors from files that valibot's own tsconfig never type-checks (its `include: ["src"]` excludes them). This is a Strata-side gap, not a genuine TS error in valibot's src/. Recorded in decisions.md.

#### Diagnostic categorization

| Category | Count | Classification |
|---|---|---|
| `Cannot find module 'vitest'` (from `*.test.ts`, `*.test-d.ts`) | 526 | Strata-side gap — test files should not be in validate() rootNames |
| `Cannot find module './dist/index.mjs'` (`playground.ts`) | 1 | Strata-side gap — root-level playground file included, not in valibot's tsconfig scope |
| `Cannot find module 'tsdown'` (`tsdown.config.ts`) | 1 | Strata-side gap — tool config not in valibot's tsconfig scope |
| `Default exports can't be inferred with --isolatedDeclarations` | 2 | Strata-side gap — `tsdown.config.ts` + `vitest.config.ts` outside corpus tsconfig scope |
| `Binding element 'format' implicitly has an 'any' type` (`tsdown.config.ts`) | 1 | Strata-side gap — tool config outside corpus tsconfig scope |

## Summary

**Verdict:** Corpus is ready for the keyed L2.5 paired comparison, with one explicit caveat: the keyed dogfood session must use the agent path (`commitWithBehavioralGate` → `tscNoEmitSrc`), not bare `commit()`. The agent path is the L2.5 dogfood's actual surface and already filters to `src/` files; the 531-diagnostic result from bare `commit()` is a Strata-side scoping gap in `validate()`, not a problem in valibot's source.

### 1. Did Strata ingest valibot/library?

YES. 1,087 modules persisted, 27 MB DB, 122,256 nodes, zero per-file errors. Ingest completed in 14.74s wall time. The 1,087 module count includes 566 source files (in `src/`, non-test), 272 `.test.ts` files, 245 `.test-d.ts` files, and 4 root-level tool configs — all ingested because `ingest-batch` does not discriminate by file role. SQL sanity counts are consistent with a real TypeScript corpus (Identifier-dominated, proportionate declaration kinds). Estimated L1 context is ~15,700 tokens, well above the >5k threshold for L2.5.

### 2. Did `find_declarations` work?

Not initially — this prep surfaced a pre-existing Strata bug. The first attempt returned 0 hits on JSDoc'd declarations because the name-extraction logic was looking for the declaration name inside the JSDoc text offset rather than at the correct AST position. The bug was diagnosed and fixed across 6 store call sites in commits `a2e19b3`, `f752671`, `90961fc`, `d4f3fcf`, `16671a1` (informed by a Codex xhigh independent review). After the fix and a store rebuild, re-verification (commit `77b98de`) confirmed:

| Query | Hits | Parent path |
|---|---|---|
| `parse` | 1 | `library/src/methods/parse/parse.ts` |
| `_isLuhnAlgo` | 1 | `library/src/utils/_isLuhnAlgo/_isLuhnAlgo.ts` |
| `__definitely_not_a_symbol__` | 0 | n/a |

Tool sanity passes. The primary L2.5 target `_isLuhnAlgo` is reachable via `find_declarations`.

### 3. Did the no-op commit gate succeed?

Mixed result. Bare `commit(db, tx)` returns `ok: false` with 531 diagnostics. All 531 diagnostics originate from files NOT covered by valibot's `include: ["src"]` — 526 "cannot find module 'vitest'" from `*.test.ts` / `*.test-d.ts`, plus 5 from root-level `playground.ts`, `tsdown.config.ts`, and `vitest.config.ts`. Zero diagnostics from `src/` files.

The root cause is a Strata-side scoping gap: `validate()` inside `commit()` passes `rootNames: [...renderedFiles.keys()]` which includes all ingested modules rather than scoping to corpus-tsconfig-covered files. This is recorded in decisions.md (commit `0e49c9d`). The gap is deferred; it only affects codebases that (a) ingest test/tool files and (b) have no `vitest` / `tsdown` installed in their node_modules at the Strata process level.

The agent-path commit gate (`commitWithBehavioralGate` → `tscNoEmitSrc`) is unaffected — it filters rendered paths to `src/` before invoking `tsc --noEmit`. This is the L2.5 dogfood's actual surface.

### 4. Is the corpus ready for the keyed L2.5 paired comparison?

YES, with the caveat stated in the verdict above. The structural surface (ingest, `find_declarations`, agent-path commit gate) all work on valibot at scale. The primary L2.5 task target (`_isLuhnAlgo` Luhn-algorithm rename) is reachable. Corpus DB is at `/tmp/valibot.strata.db` and can be regenerated from `~/code/valibot-l2.5/library` at any time using the `ingest-batch` command in Task 2's notes.

### 5. What got found that wasn't in the original plan?

Two genuine Strata findings surfaced during smoke:

**Finding 1 — `find_declarations` JSDoc-offset bug (fixed in-session):** The name-extraction logic in `packages/store` was resolving declaration names by scanning for the first `Identifier` child node at the earliest byte offset, which is the first token inside a JSDoc comment when the declaration has one. Fixed across 6 call sites by switching to a dedicated `resolveDeclarationNameIdentifier` function that selects the Identifier child whose offset falls after the JSDoc region. Commits: `a2e19b3` (Fix-A), `f752671` (Fix-A polish), `90961fc` (Fix-B), `d4f3fcf` (Fix-B polish), `16671a1` (JSDoc guard added to `examples/medium User`). The bug would have silently broken any `find_declarations` or `rename_declaration` call on a declaration with a leading JSDoc comment — a substantial fraction of a well-documented corpus like valibot.

**Finding 2 — `validate()` non-src scope mismatch (deferred):** `commit()` runs `validate()` against all ingested modules including test files and root-level tool configs, which causes spurious tsc errors on external corpora that don't have test-framework types available in the Strata process's node_modules. Recorded in decisions.md (commit `0e49c9d`). Deferred because the agent path (`commitWithBehavioralGate`) already works correctly; the bare `commit()` path is not the agent's surface. Fix when the bare path is used by anything other than smoke scripts.

### 6. Forward-looking notes for the keyed dogfood

The smoke proved the structural surface works on a real >1k-module corpus. The actual keyed paired comparison still needs a `dogfood:l2` harness (analogous to `dogfood:l1` / `dogfood:l3` in `packages/bench`) that:

- Takes a `--corpus` path pointing at the valibot `library/` directory and an optional `--db` for the pre-built store.
- Runs the primary task (P-Luhn: rename `_isLuhnAlgo` → `isLuhnFormula` across all call sites) against both the Strata agent and a file-tools baseline on a temp copy of the corpus.
- Scores both through the shared `@strata/verify` text-match criteria (does the rename propagate correctly to all references?).
- Reports token cost, tool-call count, wall time, and success/failure.

No such harness exists yet. This is the natural next workstream after Task 6. The prep scripts (`scripts/dogfood/l2.5-prep-find-declarations.ts`, `scripts/dogfood/l2.5-prep-noop-commit.ts`) can be re-run against the pre-built DB to confirm the structural surface is still clean before spending on the keyed round.

**Script placement decision:** Moved both one-off scripts from `scripts/` root to `scripts/dogfood/` (via `git mv`) to signal their reusable-but-one-off nature and keep the scripts root uncluttered. The keyed L2.5 session can re-run them directly against the pre-built DB without rebuilding anything.
