# L2.5 dogfood prep (valibot) — design

**Date:** 2026-05-27
**Status:** approved, prep work not yet executed
**Roadmap line:** `docs/product-roadmap.md` § "Iteration 2.5", unchecked `L2.5 dogfood`
**Plan reference:** `docs/specs/2026-05-26-three-layer-codebase-index-plan.md:126` (L2.5 — Dogfood comparison)

## Goal

Stand up everything required so a future session can run the L2.5 paired dogfood (`semantic_search` on vs. off) on a real, large external TypeScript corpus without per-run friction. This session does **not** run the paired comparison; the paired comparison is keyed, operator-only, and follows after the smoke confirms the corpus is usable.

L2.5 is the only unvalidated cell of the three-layer codebase index. L1 has been dogfooded (62.8% cost ratio, PASS) and L3 has been dogfooded (mechanism works; signal swamped by small corpus). L2's discovery-aid value can only be measured on a corpus where L1's name-only module index is too big to be cheap and too coarse to find the right symbol.

## Acceptance for L2.5 (recap from the index plan)

> `semantic_search-on` cost ≤ `index-on` (semantic_search-off) cost on a corpus where Layer 1 alone would inject >5k tokens.

The threshold (>5k L1 tokens) is the corpus-selection driver below. The comparison itself happens in a follow-up session.

## Corpus

`valibot/library` from `https://github.com/fabian-hiller/valibot`, single-package subtree `library/`.

Why this corpus:

- **811 TS modules in `library/src/`** (verified by file count, excluding tests). Comfortably clears the >200-module / >5k-L1-token bar.
- **Clean `library/tsconfig.json`** with `include: ["src"]` and no `extends` into the workspace root. Closest known-working shape to `unjs/defu`, which Strata already dogfooded end-to-end.
- **Discovery-by-behavior is genuinely needed.** valibot has 80+ named actions in `actions/` and a `utils/` directory of underscore-prefixed internal helpers (`_isLuhnAlgo`, `_getGraphemeCount`, `_joinExpects`, `_addIssue`, `_cloneDataset`, …). Name-only L1 retrieval cannot find these by behavior; semantic_search has a real job to do.
- **No exotic build / codegen.** Pure TypeScript, modular, JSR-published.

Other candidates rejected during selection:

- **arktype** — package sources live at the package root with no `src/` directory; conflicts with Strata's strict-`src/`-only tsconfig assertion (per `decisions.md` defu fix). Also smaller than expected per package (`ark/type` ≈ 55, `ark/schema` ≈ 51).
- **date-fns** (`pkgs/core/src` ≈ 1494 modules) — extends `@date-fns/dev` (workspace dep, would fight Strata); module names are the function names, weakening the semantic_search story.
- **hono** (`src/` ≈ 186 modules) — sits at the threshold; tsconfig uses project references, so Strata would need the build-variant.
- **zod** (`packages/zod/src` ≈ 118 modules) — below threshold; tsconfig uses custom conditions.

## Smoke sequence (unkeyed)

All of the following must complete cleanly before any L2 measurement is considered. Each step is a falsifier — failure surfaces a Strata gap, not an L2 finding.

1. `pnpm --filter @strata/cli build` — baseline CLI freshness.
2. `node packages/cli/dist/cli.js ingest-batch ~/code/valibot-l2.5/library /tmp/valibot.strata.db` — ingest completes without panic; DB persists.
3. **Sanity SQL** against `/tmp/valibot.strata.db`: row counts on `nodes`, `module_index`, and `Identifier`. Confirm order-of-magnitude — `module_index` should be ≈ 811; `nodes` and `Identifier` should scale accordingly.
4. **Tool sanity check:** invoke `find_declarations` for a known symbol (e.g. `parse`) via a small node harness against the in-process tools (no LLM call required). Confirm it returns sensible results.
5. **Commit gate sanity:** open a no-op transaction (begin → commit with zero ops) and confirm the gate runs clean against the rendered corpus.

**Acceptance:** all five steps pass.

## Embedding smoke (keyed, optional this session)

Only if `STRATA_EMBED_API_KEY` is set and the operator chooses to spend ~$0.04 of embedding cost (per the index plan's risk estimate):

6. `STRATA_EMBED_API_KEY=… node packages/cli/dist/cli.js embed ~/code/valibot-l2.5/library --db /tmp/valibot.strata.db`
7. Sanity SQL: `SELECT COUNT(*) FROM node_embeddings` ≈ count of top-level declarations. Record the embedding wall time and cost; this is the amortized one-time cost the L2.5 acceptance explicitly excludes from the paired comparison.

If this step is skipped, the paired comparison in the follow-up session simply runs it before measuring.

## Dogfood task design

The L2.5 comparison is paired: same task, same model, same prompt, run twice — once with `semantic_search` available (L1+L2) and once without (L1 only). For the measurement to isolate L2, the task must satisfy:

- **Discovery-heavy.** Most of the cost is the agent finding the right symbol, not mutating it.
- **Discovery-by-behavior.** The agent is given a description of what the helper does, not its name. L1's module index alone (which is name-only) is not enough.
- **Mutation present but trivial.** A small mutation lets the commit gate verify the agent found the *right* symbol (not a plausibly-named decoy), without inflating cost.

Three candidate prompts, all targeting underscore-prefixed internal helpers in `library/src/utils/`:

| ID | Prompt | Target |
|----|--------|--------|
| **P-Luhn** | *"Find the internal helper that validates credit-card numbers using the Luhn algorithm. Add a one-line JSDoc above its declaration explaining what it does."* | `_isLuhnAlgo` |
| P-Graphemes | *"Find the internal helper that counts grapheme clusters in a string. Add a one-line JSDoc above its declaration explaining what it does."* | `_getGraphemeCount` |
| P-JoinExpects | *"Find the internal helper that joins expected-value strings for error messages. Add a one-line JSDoc above its declaration explaining what it does."* | `_joinExpects` |

**Primary: P-Luhn.** Reasons: the Luhn algorithm is unambiguous so the agent can self-verify by reading five lines of code; the behavior description shares almost no surface terms with the symbol name `_isLuhnAlgo`; the mutation is a comment-only edit, minimizing commit-gate noise.

P-Graphemes and P-JoinExpects are held as backups in case P-Luhn turns out to be too easy (e.g. some text in the surrounding file already says "Luhn", letting a grepping agent stumble onto it).

## Open questions and risks

- **JSDoc-only mutation through the commit gate.** Adding a JSDoc comment above a declaration is a `replace_body`-shaped or `replace_declaration`-shaped operation. Whether Strata's current tools express this cleanly is not yet known. If the commit gate fights, the fallback is a `// purpose: …` line inside the function body (a body-replace, more proven path), or a pure-discovery harness where the agent just reports the module path. Smoke step 4/5 will surface this.
- **Ingest at scale.** Strata has so far only ingested `examples/medium` and defu. 811 modules with valibot's heavy generics / conditional types may surface a real ingest gap. That's a valuable finding independent of L2, and should be logged in `decisions.md` if it happens.
- **Don't conflate ingest success with L2 win.** "Ingest works" and "L2 beats L1-only" are separate questions. Only the first is in scope this session.
- **L1 token cost on valibot is not yet measured.** The >5k threshold is asserted by module count, not directly measured. The first paired run will produce that number; it should be recorded.
